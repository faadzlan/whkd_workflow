* Quick Start

** Loading the Workflows

Load all shortcuts into your current PowerShell session:
#+begin_src powershell
# Navigate to project
cd C:\Users\<USERNAME>\Documents\whkd_workflow

# Load all workflows (the dot is required!)
. .\tools\workflows\Load-WorkflowProfile.ps1
#+end_src

** Auto-Load on Every PowerShell Session

Add to your PowerShell profile (~/.bashrc equivalent):
#+begin_src powershell
notepad $PROFILE

# Add this line at the end:
. C:\Users\faadz\Documents\whkd_workflow\tools\workflows\Load-WorkflowProfile.ps1
#+end_src

*** Adding Your Own Projects

Edit ~tools/workflows/projects.json~ and add your project to the ~projects~ object:
#+begin_src json
{
  "projects": {
    "whkd": {
      "path": "~\\Documents\\whkd_workflow",
      "description": "Windows Hotkey Daemon setup project"
    },
    ...
    "thesis": {
      "path": "G:\\My Drive\\Academic\\Thesis",
      "description": "PhD thesis work"
    },
    "teaching": {
      "path": "G:\\My Drive\\Teaching\\2024",
      "description": "Course materials"
    }
  }
}
#+end_src

The ~proj~ command automatically picks up changes from this JSON file (single source of truth).

** System Monitoring

| Command | Description | Bash Equivalent |
|---------+-------------|-----------------|
| ~syshealth~ | Full system dashboard | ~top~ + ~free~ + ~df -h~ |
| ~mem~ | Top 10 memory consumers | ~ps aux --sort=-%mem~ |
| ~cpu~ | Top 10 CPU consumers | ~ps aux --sort=-%cpu~ |

*** syshealth Output Sections

1. *Memory Usage* - Total/Used/Free with color-coded status (Green < 75% < Yellow < 90% < Red)
2. *Top Memory Consumers* - Which processes are eating RAM
3. *CPU Usage* - Current load percentage
4. *Disk Space* - Like ~df -h~ for all drives
5. *Startup Programs* - What runs at boot
6. *Network* - IP address and connectivity status

* Individual Scripts

** Start-WorkEnvironment.ps1

Opens and positions your daily workspace.

*** What It Does
1. Switches to Desktop 2 using VirtualDesktop module
2. Opens 2 File Explorers stacked on the left
3. Opens Windows Terminal on the right half
4. Switches to Desktop 3
5. Opens Emacs (WSL) at project root (detached)
6. Returns to Desktop 2

*** Parameters

| Parameter | Type | Default | Description |
|-----------+------+---------+-------------|
| ~-ProjectPath~ | string | ~G:\My Drive\Projects~ | Project directory to open |
| ~-SkipTerminal~ | switch | ~$false~ | Don't open Windows Terminal |
| ~-SkipExplorer~ | switch | ~$false~ | Don't open File Explorers |
| ~-SkipEmacs~ | switch | ~$false~ | Don't open Emacs |

*** Examples

#+begin_src powershell
# Default usage
.\tools\workflows\Start-WorkEnvironment.ps1

# Today's specific project
.\tools\workflows\Start-WorkEnvironment.ps1 -ProjectPath "G:\My Drive\Projects\OpenFOAM\Case1"

# Just the terminal and explorers, no Emacs
.\tools\workflows\Start-WorkEnvironment.ps1 -SkipEmacs
#+end_src

** Close-AllWindows.ps1

Gracefully closes all open application windows.

*** Parameters

| Parameter | Type | Default | Description |
|-----------+------+---------+-------------|
| ~-Exclude~ | string[] | ~@("explorer", "whkd")~ | Processes to keep running |
| ~-Force~ | switch | ~$false~ | Kill without asking |

*** Examples

#+begin_src powershell
# Normal usage (graceful close)
.\tools\workflows\Close-AllWindows.ps1

# Also close explorer.exe
.\tools\workflows\Close-AllWindows.ps1 -Exclude @("whkd")

# Force kill everything (except whkd)
.\tools\workflows\Close-AllWindows.ps1 -Force
#+end_src

** Clear-Downloads.ps1

Archives old Downloads files to prevent Explorer hangs.

*** How It Works
- Finds files older than N days (default: 30)
- Moves them to ~Downloads\_Archive\YYYY-MM-DD\~
- Preserves folder structure
- Removes empty directories

*** Parameters

| Parameter | Type | Default | Description |
|-----------+------+---------+-------------|
| ~-DaysOld~ | int | ~30~ | Archive files older than this |
| ~-AutoArchive~ | switch | ~$false~ | Skip confirmation prompt |
| ~-WhatIf~ | switch | ~$false~ | Preview only, don't move |

*** Examples

#+begin_src powershell
# Interactive mode (shows what will be moved, asks confirmation)
.\tools\workflows\Clear-Downloads.ps1

# Preview what would be archived
.\tools\workflows\Clear-Downloads.ps1 -WhatIf

# Archive files older than 7 days without asking
.\tools\workflows\Clear-Downloads.ps1 -DaysOld 7 -AutoArchive
#+end_src

*** Scheduling

Run weekly via Task Scheduler:
#+begin_src powershell
schtasks /create /tn "Weekly Downloads Cleanup" /tr "powershell.exe -File `"C:\Users\faadz\Documents\whkd_workflow\tools\workflows\Clear-Downloads.ps1`" -AutoArchive" /sc weekly /d SUN /st 18:00
#+end_src

* Troubleshooting

** "Command not found" after loading profile

Make sure you used the *dot*:
#+begin_src powershell
# WRONG - runs in separate scope, functions disappear
.\tools\workflows\Load-WorkflowProfile.ps1

# RIGHT - sources into current scope
. .\tools\workflows\Load-WorkflowProfile.ps1
#+end_src

** VirtualDesktop module not working

#+begin_src powershell
# Check if installed
Get-Module -ListAvailable -Name VirtualDesktop

# Install if missing
Install-Module -Name VirtualDesktop -Force -Scope CurrentUser
#+end_src

** Get-ChildItemColor module not working

The ~proj~ command uses Get-ChildItemColor for colorized directory listings.

#+begin_src powershell
# Check if installed
Get-Module -ListAvailable -Name Get-ChildItemColor

# Install if missing
Install-Module -Name Get-ChildItemColor -Scope CurrentUser
#+end_src

** Emacs doesn't open

Check that your ~emc~ function works:
#+begin_src powershell
# Test directly
emc

# If that works, the script should work too
#+end_src

** Window positioning not working

Window positioning requires the window to be ready. If windows aren't moving:
1. Close other heavy applications
2. Increase the ~Start-Sleep~ delays in the script
3. Run the script again
